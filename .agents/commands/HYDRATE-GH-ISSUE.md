# Hydrate Github Issue Details v1.0.0

## Overview

Use the Github MCP server (or the Github (gh) skill) to create a description for the provided Github issue (if a URL was provided, update the issue, otherwise create a new issue).

This should capture the high level requirements and details of the request.

This should be a detailed description of the request, including the expected behavior, and any relevant context without getting into the details of the implementation with code.

One-at-a-time, ask for clarification on any details you are not 100% sure about.  The options should be listed as A, B, C, to make choosing easier.
