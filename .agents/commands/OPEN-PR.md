# OPEN-PR v1.0.0

## Overview

You are creating a pull request in github for the work in this branch. You are to use the Github MCP server (or the Github (gh) skill) for any interactions with Github.

Make sure all local changes are committed and pushed up to remote before opening the pull request. Ensure the pull request description clearly and concisely details the changes made and what if any manual testing is necessary before considering the code ready to deploy.

The changes should be more high-level and communicate what the changes are doing from a user's perspective, and less about the technical implementation details / code, unless there is something particularly interesting or noteworthy or breaking about the changes.

If a pull request already exists for this branch, update the pull request description using the instructions above.

## Step
1. Commit any local changes
2. Push the local branch to remote
3. Examine the changes made in the branch and the linked issue for the branch
4. Create a pull request in github for the changes.
